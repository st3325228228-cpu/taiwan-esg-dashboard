# Taiwan ESG Dashboard 

使用 HTML、CSS、JavaScript、Chart.js 與 Plotly.js 建立的台灣上市公司溫室氣體排放資料視覺化儀表板。

本專案以企業溫室氣體排放資料為主題，呈現範疇一、範疇二、範疇三排放量、排放密集度、產業分布與企業排行，並透過 GitHub Pages 提供可直接操作的靜態網站。

目前版本使用內建於 JavaScript 的示範資料建立前端儀表板，主要展示 ESG 資料結構、互動式圖表、響應式網頁設計與靜態網站部署能力。

## 重要資料說明

本專案介面以證交所 MOPS Open Data 的溫室氣體排放資料格式與 ESG 指標為設計參考。

但目前 GitHub Pages 版本所顯示的公司資料與 KPI 數值，是直接寫在 `index.html` JavaScript 中的靜態示範資料，不是每次開啟頁面時即時呼叫 MOPS API。

因此，本專案目前的定位是：

```text
ESG 資料視覺化前端作品
```

而不是：

```text
即時 MOPS 資料查詢系統
```

介面中的企業名稱、排放量、排放密集度、公司數量與其他數值，主要用於展示圖表及互動功能，不應直接作為正式研究、企業比較或投資判斷依據。

## 儀表板內容

### 1. 範疇一總排放

顯示企業直接溫室氣體排放摘要。

範疇一通常指企業擁有或控制來源所產生的直接排放，例如：

- 工廠燃料燃燒
- 生產製程排放
- 公司車輛燃料使用
- 冷媒逸散
- 其他企業直接控制的排放來源

目前畫面中的總排放量為靜態展示數值。

### 2. 範疇二總排放

顯示企業因購買能源而產生的間接排放摘要。

常見來源包括：

- 外購電力
- 外購蒸氣
- 外購熱能
- 外購冷卻能源

目前畫面中的總排放量為靜態展示數值。

### 3. 範疇三總排放

顯示企業價值鏈所產生的其他間接排放摘要。

可能包括：

- 原物料採購
- 供應商排放
- 商品運輸
- 員工通勤
- 商務旅行
- 產品使用
- 廢棄物處理
- 其他上下游活動

目前畫面中的總排放量為靜態展示數值。

### 4. 平均排放密集度

排放密集度用於比較企業每單位營收對應的溫室氣體排放量。

本專案介面使用的單位為：

```text
噸 CO₂e／百萬元營收
```

排放密集度可協助比較不同規模企業的相對碳排放程度。

但目前數值為示範資料，不應直接作為正式企業績效評估。

### 5. 公司數量

儀表板顯示資料庫公司總數，用於模擬完整上市公司 ESG 資料庫的摘要資訊。

目前此數值是介面中的固定展示資料，不代表程式實際載入相同數量的企業紀錄。

### 6. 最高排放企業

顯示示範資料中範疇一排放量最高的企業，協助使用者快速辨識高排放公司。

---

## 範疇定義

| 範疇 | 名稱 | 說明 |
|---|---|---|
| Scope 1 | 直接排放 | 由企業擁有或控制的排放來源直接產生 |
| Scope 2 | 能源間接排放 | 因企業購買電力、蒸氣、熱能或冷卻能源所產生 |
| Scope 3 | 其他間接排放 | 企業上下游價值鏈及其他非直接控制活動產生 |

## 公司範圍控制

使用者可透過滑桿設定要顯示的公司數量。

目前設定範圍：

```text
最少：5 家
最多：50 家
調整間隔：5 家
預設：10 家
```

調整完成後，點選：

```text
更新圖表
```

系統會重新產生：

- 範疇一排放占比圖
- 三大範疇總量圖
- 企業排放比較圖
- 排放密集度趨勢圖
- 產業旭日圖
- 排放量排行榜

## 圖表功能

### 1. 範疇一排放占比圓餅圖

使用 Doughnut Chart 顯示前 N 家企業的範疇一排放占比。

可觀察：

- 哪些企業的直接排放量較高
- 高排放企業占整體的比例
- 排放量是否集中於少數企業
- 不同企業間的相對差異

滑鼠移到圖表區塊時，可查看：

- 股票代號
- 公司名稱
- 範疇一排放量

### 2. 三大範疇總量比較圖

使用圓餅圖比較：

- 範疇一直接排放
- 範疇二能源間接排放
- 範疇三其他間接排放

此圖表可快速呈現不同排放範疇的相對占比。

目前總量數值為固定展示資料。

### 3. 前 N 家企業排放量柱狀圖

使用群組柱狀圖比較各企業的：

- 範疇一排放
- 範疇二排放
- 範疇三排放

可觀察：

- 不同企業的排放結構
- 哪一類排放占主要比例
- 高排放與低排放企業差距
- 不同企業範疇三排放是否明顯

### 4. 排放密集度折線圖

使用折線圖顯示前 N 家企業的排放密集度。

圖表單位：

```text
噸 CO₂e／百萬元營收
```

可協助觀察：

- 各企業排放密集度差異
- 高排放密集度企業
- 企業排放效率的相對位置
- 排放總量與排放密集度的差異

### 5. 產業與企業旭日圖

使用 Plotly Sunburst Chart 呈現階層資料。

目前階層結構：

```text
產業
  └── 公司
        └── 範疇一排放量
```

產業分類示例包括：

- 水泥
- 鋼鐵
- 石化
- 汽車
- 零售
- 金融
- 半導體
- IC 設計
- 電腦
- 電信
- 記憶體
- 電子
- 其他產業

使用者可以點擊旭日圖節點，展開或返回不同層級。

### 6. 排放量排行榜

排行榜依照範疇一排放量由高至低排序。

資料欄位包括：

- 排名
- 股票代號
- 公司名稱
- 範疇一排放量
- 範疇二排放量
- 範疇三排放量
- 排放密集度
- 範疇一排放占比

排行榜可協助使用者快速比較前 N 家企業。

## 資料結構

目前 JavaScript 示範資料的主要欄位如下：

```javascript
{
  code: "1101",
  name: "台泥",
  s1: 4821,
  s2: 312,
  s3: 1840,
  intensity: 142.3
}
```

### 欄位說明

| 欄位 | 說明 |
|---|---|
| `code` | 股票代號 |
| `name` | 公司名稱 |
| `s1` | 範疇一排放量 |
| `s2` | 範疇二排放量 |
| `s3` | 範疇三排放量 |
| `intensity` | 排放密集度 |

目前企業資料依照範疇一排放量排序，並由前端程式根據使用者選擇的 Top N 數量截取。

## 本機執行

本專案不需要安裝 Node.js、Python 套件或其他相依套件。

但為了避免瀏覽器對本機檔案的限制，建議使用本機 HTTP 伺服器開啟。

### 方法一：使用 Python

在專案資料夾執行：

```bash
python -m http.server 8000
```

接著開啟：

```text
http://localhost:8000
```

### 方法二：使用 Visual Studio Code

1. 安裝 Live Server 擴充套件。
2. 開啟 `index.html`。
3. 點選 `Open with Live Server`。
4. 在瀏覽器查看儀表板。

### 方法三：直接開啟

也可以直接雙擊：

```text
index.html
```

但部分 CDN、瀏覽器安全設定或外部資源可能無法正常載入，因此仍建議使用 HTTP 伺服器。

## 目前限制

- 目前使用靜態示範企業資料。
- 不會在頁面開啟時即時呼叫 MOPS API。
- 介面顯示的「即時數據」文字與目前實作不完全一致。
- KPI 卡片中的部分數值是固定展示值。
- 顯示的公司總數不等於 JavaScript 中實際存在的公司筆數。
- 資料年份顯示為 2024 年報，但不代表已自動更新。
- 公司資料可能不是完整上市公司名單。
- 產業分類為前端程式中的手動對照表。
- 排放密集度為示範數值。
- 無法搜尋特定公司。
- 無法按照產業或排放範疇篩選。
- 無法下載 CSV。
- 無法切換資料年度。
- 無後端資料庫。
- 無使用者登入或權限管理。
- 無資料更新排程。
- 外部 CDN 中斷時可能影響圖表顯示。
- 本專案不適合直接用於正式 ESG 稽核或投資分析。
